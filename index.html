It is possible since with Kotlin and Swift the machine itself can understand which type of data is it — whether it is Int or String, for example. When the machine do so it is called type inference.

Type inference is done automatically under the hood — in other words, without our participation and only by the machine itself.

Anyways, type inference is one of reasons why people around the world love Kotlin and Swift so much. Type inference makes the coder's life a lot easier. Simultaneously, it makes code more clear and concise, meaning awesome.

Your Own Slave Commanders
 You already know a lot of awesome things about Commanders:


Each Commander has its name and makes the machine do some specific work.

There are various sorts of Commanders.

You already know a lot about Pocket Commanders, Arithmetic Commanders, the Commander print and others.

Anyways, reality, again, is even more stunning!

You may create your own Slave Commanders:


Each your Slave Commander will be a slave to you but the commander to the machine.

When you create your Slave Commander, you can establish (describe) the specific work that this Slave Commander will make the machine do.

So your Slave Commander can make the machine do the specific work that you previously established (described).

By the way, since all Commanders are our helpers, they all in a way are our slaves. But the thing of Slave Commanders is that precisely you establish (describe) the specific work that your Slave Commander make the machine do.

Slave Commanders are called Functions.

Creating a Function happens with the word fun in Kotlin and func in Swift.

Very Interesting Objects
 We already know that in programming the number 5 or the string Hello, for example, is an object. And we can control these objects — in other words, we can do some action on these objects.

We also know that Commanders — for example, Pocket Commanders or Array Commanders — may be objects as well; and we can also do some action on these objects.

But additionally, we can also create much more interesting objects than all those ones we know for now.

Let's start diving into this unbelievably wonderful world.

Suppose that we want to write a program to record information about pets.


As we continue thinking about them, we realize that all pets are quite similar.

If we take, for example, a dog and a cat; each of them has its name, its age, its breed, its sex, its color, its way they sound, and so on.

In other words, each of them has its object on each its property:

species = "dog", name = "Max", age = 2, breed = "Beagle", sex = "male", color = "brown", sound = "bark", class = "pet"

species = "cat", name = "Kitty", age = 1, breed = "Burmilla", sex = "female", color = "white", sound = "meow", class = "pet"

As we can see, nevertheless they both even have and will always have one thing in common: each of them is a pet!

So the class pet is what they have in common — the pet is the object they have and will always have in common.

An example of code in Kotlin: 

class pet(
    val anySpecies: String,
    val anyName: String,
    val anyAge: Int,
    val anyBreed: String,
    val anySex: String,
    val anyColor: String,
    val anySound: String
)
{
	init {
    	println("The $anySpecies $anyName has been created!")
        println("It can $anySound!")
        println("$anyName is a $anySex, it is $anyColor and $anyAge.")
        println("Its breed is $anyBreed.")
        println(" ")       
    }
}

fun main() {
    val myDog = pet("dog", "Max", 2, "Beagle", "male", "brown", "bark")
    val myCat = pet("cat", "Bella", 1, "Burmilla", "female", "white", "meow")
}

The dog Max has been created!
It can bark!
Max is a male, it is brown and 2.
Its breed is Beagle.

The cat Bella has been created!
It can meow!
Bella is a female, it is white and 1.
Its breed is Burmilla.
