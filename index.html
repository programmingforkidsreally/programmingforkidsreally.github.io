Learn from Code
 Please try learning from code.

In order to do so just imagine a situation in which you would not have any description (explanation) to code and should learn everything only from the code itself:
· analyzing the code and the result,
· playing around with the code.

Learning programming this way, step by step, is called learning from code.

This approach may be very effective.

So you may practice the most extreme approach here: If some code is clear to you, ignore all of my explanations to it.

Try learning from code as much as possible.

Only 2 types of objects
 Take a look at this familiar code:

  fun main() {
    val Sam = 5
    print(Sam)
}
let Sam = 5
print(Sam)
 
         
  5   5  
  Try It Yourself   Try It Yourself  

Here we give the machine — as an object — the number 5.

Yet another familiar code:

  fun main() {
    val Sam = "Hello"
    print(Sam)
}
let Sam = "Hello"
print(Sam)
 
         
  Hello   Hello  
  Try It Yourself   Try It Yourself  

Here we give the machine — as an object — the string Hello.

As we can see from the codes above, we can give the machine — as an object — data of certain type: the number 5 or the string Hello, for example.

Indeed, there are a more number of data types that we might give the machine. You yourself can do your own searching over this topic on the Internet.

We here will only take a look at the 2 data types:
· Int — which might be represented by the number 5, for example;
· String — which might be represented by the string Hello, for example.



The Int here stands for the integer number.

Absolutely, manipulating just these 2 data types — Int and String — are more than enough to get an idea of the entire data type concept in programming.

This Works Under the Hood


On the Earth, there are lots of stupid programming languages in which we always must explicitly state the type of the object: whether Int or String, for example.

An example of code:

  fun main() {
    val Sam: Int = 5
    print(Sam)
}
let  Sam: Int = 5
print(Sam)
 
         
  5   5  
  Try It Yourself   Try It Yourself  

Yet another example of code:

  fun main() {
    val Sam: String = "Hello"
    print(Sam)
}
let Sam: String = "Hello"
print(Sam)
 
         
  Hello   Hello  
  Try It Yourself   Try It Yourself  

Fortunately, in Kotlin and Swift, as we already know, we can, if wanted, not state the type of the object since it may be done automatically. Automatically, by the machine itself!

It is called automatic type inference, or just type inference.

Anyways, type inference is one of reasons why people around the world love Kotlin and Swift so much. Automatic type inference makes code more clear and concise, and it means, awesome.
