Only 2 types
 Take a look at this familiar code:

  fun main() {
    val Sam = 5
    print(Sam)
}
let Sam = 5
print(Sam)
 
         
  5   5  
  Try It Yourself   Try It Yourself  

Here we give the machine — as an object — the number 5.

Yet another familiar code:

  fun main() {
    val Sam = "Hello"
    print(Sam)
}
let Sam = "Hello"
print(Sam)
 
         
  Hello   Hello  
  Try It Yourself   Try It Yourself  

Here we give the machine — as an object — the string Hello.

As we can see from the codes above, we can give the machine — as an object — data of certain type: whether the number 5 or the string Hello, for example.

Indeed, there are a more number of types of data that we might give the machine. You yourself can do your own searching over this topic on the Internet.

We here will only take a look at data of the 2 types:
· Int — which might be represented by the integer number 5, for example;
· String — which might be represented by the string Hello, for example.



Of course, the Int here stands for the integer number.

Absolutely, juggling just these 2 data types — Int and String — are enough to get an idea of the entire data type concept in programming.




This Works Under the Hood

On the Earth, there are lots of stupid programming languages in which we always must explicitly write (report, state) the type of data we give the machine — whether it is Int or String, for example.

Fortunately, in Kotlin and Swift this reporting is only optional.

An example of code with reporting:

  fun main() {
    val Sam: Int = 5
    print(Sam)
}
let  Sam: Int = 5
print(Sam)
 
         
  5   5  
  Try It Yourself   Try It Yourself  

Yet another example of code with reporting:

  fun main() {
    val Sam: String = "Hello"
    print(Sam)
}
let Sam: String = "Hello"
print(Sam)
 
         
  Hello   Hello  
  Try It Yourself   Try It Yourself  

Thankfully, in Kotlin and Swift we can, if wanted, not write (not report, not state) the type of data we give the machine — whether it is Int or String, for example.

An example of code without reporting:

  fun main() {
    val Sam = 5
    println(Sam)

    val Leo = "Hello"
    print(Leo)
}
let Sam = 5
print(Sam)

let Leo = "Hello"
print(Leo)
 
         
  5
Hello		   5
Hello		  
  Try It Yourself   Try It Yourself  

It is possible since with Kotlin and Swift the machine itself can understand which type of data is it — whether it is Int or String, for example. When the machine do so it is called type inference.

Type inference is done automatically under the hood — in other words, without our participation and only by the machine itself.

Anyways, type inference is one of reasons why people around the world love Kotlin and Swift so much. Type inference makes the coder's life a lot easier. Simultaneously, it makes code more clear and concise, meaning awesome.

Glory to the Type Inference! We mean, glory to the teams behind Kotlin and Swift!

Functions
 You already know a lot of awesome things about Commanders:


Each Commander has its name and makes the machine do some specific work.

There are various sorts of Commanders.

You already know a lot about Pocket Commanders, Arithmetic Commanders, the Commander print and others.

Anyways, reality, again, is even more stunning!

We may create our own free Commanders:


The word free there means that these Commanders can make the machine do anything we say to them. We are free about our desires to say to them anything.

So, thanks to these Commanders, we can really make the machine (computer, robot) do whatever we want. The imagination is the limit!

Such Commanders are called Functions.

To make that magic, first of all, we have to create a Function. And then, we may call (invoke) this Function.

Let's create the Function named displayThisName().

Take a look at this code:
  

fun displayThisName(anyName: String) {
    print(anyName)
}
fun main() {
    displayThisName("Dave")
}

Dave
